# Netflix Recommender System

A collaborative filtering recommender system built on the Netflix Prize dataset, comparing **SVD (Matrix Factorisation)** against **Item-Based Collaborative Filtering (KNN)** using RMSE, MAE, and MAP@10.

---

## Requirements

Python 3.8 or higher is recommended.

Install all dependencies with:

```bash
pip install pandas numpy matplotlib seaborn scikit-surprise tqdm scikit-learn jupyter
```

| Package | Purpose |
|---|---|
| pandas, numpy | Data loading and manipulation |
| matplotlib, seaborn | EDA visualisations |
| scikit-surprise | SVD and KNN collaborative filtering models |
| tqdm | Progress bars during file parsing |
| scikit-learn | Precision, Recall, NDCG metric computation |
| jupyter | Running the models notebook |

## Dataset

Download the Netflix Prize dataset from [Kaggle](https://www.kaggle.com/datasets/netflix-inc/netflix-prize-data) and place the following files in the `data/` folder:

- `combined_data_1.txt`
- `movie_titles.csv`

> Only `combined_data_1.txt` is used in this project (approx. 24 M ratings). The full dataset includes four combined data files.

## How to Reproduce Results

Run the four steps in order from the project root directory.

### Step 1 — Load Data

Parses the raw Netflix ratings file and creates a working subset of the 10,000 most active users.

```bash
python load_data.py
```

**Output:** `data/netflix_subset.csv`, `data/movies_clean.csv`

---

### Step 2 — Clean Data

Removes duplicates, filters low-activity users (< 5 ratings) and rarely-rated movies (< 10 ratings), fixes data types, and adds derived date columns.

```bash
python clean_data.py
```

**Output:** `data/netflix_clean.csv` (overwrites movies_clean.csv in place)

---

### Step 3 — Exploratory Data Analysis

Generates all 8 EDA plots and prints key dataset statistics.

```bash
python eda.py
```

Optional arguments:

```bash
python eda.py --ratings data/netflix_clean.csv --movies data/movies_clean.csv --plots-dir plots
```

**Output:** 8 PNG plots saved to `plots/`

| Plot | Description |
|---|---|
| 01_rating_distribution.png | Distribution of 1–5 star ratings |
| 02_user_activity.png | Ratings per user histogram |
| 03_movie_popularity.png | Ratings per movie histogram |
| 04_top20_movies.png | Top 20 most-rated movies |
| 05_top20_avg_rating.png | Top 20 highest-rated movies (min 100 ratings) |
| 06_ratings_over_time.png | Rating volume by year |
| 07_avg_rating_over_time.png | Average rating by year |
| 08_sparsity_heatmap.png | User-movie matrix sparsity visualisation |

---

### Step 4 — Train Models & Evaluate

Open and run all cells in the notebook:

```bash
jupyter notebook models.ipynb
```

The notebook covers:

1. Load clean data
2. Temporal train/test split (cutoff: 2005-10-01)
3. Build Surprise datasets
4. Train SVD (Matrix Factorisation)
5. Train Item-Based CF (KNN)
6. Compute RMSE, MAE, MAP@10
7. Generate top-10 recommendations for sample users
8. Analyse success and failure cases
9. Save models to `models/` and results to `data/model_results.csv`

**Results:**

| Model | RMSE | MAE | MAP@10 |
|---|---|---|---|
| SVD (MF) | 0.8885 | 0.6775 | 0.7767 |
| Item-Based CF | 1.0002 | 0.7730 | 0.7014 |

## Model Details

### SVD — Matrix Factorisation

Decomposes the user-item matrix into latent factor vectors. Each user and item is represented by a vector in a shared latent space. Predicted rating:

```
r-hat(u,i) = mu + b_u + b_i + p_u . q_i
```

| Hyperparameter | Value |
|---|---|
| n_factors | 50 |
| n_epochs | 20 |
| lr_all | 0.005 |
| reg_all | 0.02 |

### Item-Based CF — KNN

Recommends items similar to those a user has already rated, using cosine similarity on rating co-occurrence vectors.

| Hyperparameter | Value |
|---|---|
| k (max neighbours) | 40 |
| min_k | 5 |
| similarity | Cosine |
| user_based | False |

## Notes

- `data/` is excluded from version control. Add `data/` to your `.gitignore` — the raw Netflix files are several GB and redistribution is not permitted under the dataset licence.
- Trained models are saved as `models/svd.pkl` and `models/ibcf.pkl`. These can be large; consider excluding `models/` from git and regenerating by re-running the notebook.
- The technical report is saved to `reports/netflix_recsys_report.pdf` after running the full pipeline.
- Training SVD on 3.5 M ratings takes approximately 90 seconds on a modern CPU.
